## 1. Vite: path alias, Rollup `manualChunks`, Terser production cleanup

**Files:** `vite.config.ts`

```ts
import path from 'node:path'
import { defineConfig } from 'vite'

export default defineConfig({
  resolve: {
    alias: { '@': path.resolve(__dirname, './src') },
  },
  build: {
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true,
      },
    },
    rollupOptions: {
      output: {
        manualChunks(id) {
          if (!id.includes('node_modules')) return
          if (id.includes('@base-ui')) return 'base-ui'
          if (id.includes('zustand')) return 'zustand'
          if (id.includes('node_modules/react-dom')) return 'react-vendor'
          if (id.includes('node_modules/react/')) return 'react-vendor'
        },
      },
    },
    chunkSizeWarningLimit: 600,
  },
})
```

**English:** `@/` imports keep imports short and stable. `manualChunks` splits vendor code so browsers cache React and UI libraries separately and download them in parallel. Terser removes `console` and `debugger` in production builds. The `terser` package must be installed when using `minify: 'terser'`.

**العربية:** مسار الاسم المستعار `@/` يُسهّل الاستيراد ويُثبّت المسارات. `manualChunks` يقسّم حزم `node_modules` (مثل React و`@base-ui`) لتخزين مؤقت أفضل وتحميل متوازٍ. Terser يحذف `console` و`debugger` في الإنتاج. يجب تثبيت حزمة `terser` عند استخدام `minify: 'terser'`.

## 3. Anti-FOUC theme script + Zustand `persist` (same storage key)

**Files:** `index.html`, `src/stores/theme-store.ts`

```html
<script>
  (function () {
    var key = 'currency-theme'
    var fallback = 'dark'
    try {
      var raw = localStorage.getItem(key)
      var theme = fallback
      if (raw) {
        var p = JSON.parse(raw)
        if (p && p.state && (p.state.theme === 'light' || p.state.theme === 'dark'))
          theme = p.state.theme
      }
      document.documentElement.classList.toggle('dark', theme === 'dark')
    } catch (e) {
      document.documentElement.classList.add('dark')
    }
  })()
</script>
```

```ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

export const THEME_STORAGE_KEY = 'currency-theme'

export const useThemeStore = create(
  persist(
    (set, get) => ({
      theme: readThemeFromStorage(),
      setTheme: (theme) => {
        set({ theme })
        applyThemeToDocument(theme)
      },
      toggleTheme: () => {
        const next = get().theme === 'dark' ? 'light' : 'dark'
        get().setTheme(next)
      },
    }),
    {
      name: THEME_STORAGE_KEY,
      partialize: (state) => ({ theme: state.theme }),
      onRehydrateStorage: () => (state) => {
        if (state?.theme) applyThemeToDocument(state.theme)
      },
    }
  )
)
```

**English:** The inline script runs **before** React paints, reading the same persisted Zustand shape (`state.theme`) from `localStorage` and toggling the `.dark` class on `<html>`. That avoids a **flash of wrong theme** (FOUC). `persist` keeps the theme after refresh; `partialize` stores only `theme`; `onRehydrateStorage` reapplies the class after rehydration.

**العربية:** السكربت في `<head>` يعمل **قبل** رسم React ويقرأ نفس مفتاح التخزين وبنية Zustان المخزّنة (`state.theme`) ويضبط class الوضع الداكن على `<html>` لتجنب **وميض لون خاطئ** عند التحميل. `persist` يحفظ السمة بعد التحديث؛ `partialize` يخزّن حقل السمة فقط؛ `onRehydrateStorage` يعيد تطبيق الـ class بعد إعادة التحميل من التخزين.

## 4. `React.lazy` + `Suspense` + deferred viewport loading

**Files:** `src/components/currency-landing/CurrencyLanding.tsx`, `DeferredSection.tsx`

```ts
import { lazy } from 'react'

const ServicesSection = lazy(() =>

export function CurrencyLanding() {
  return (
    <>
      <HeroSection />
      <DeferredSection>
        <ServicesSection />
      </DeferredSection>
    </>
  )

```

```tsx
import { memo, Suspense, type ReactNode } from 'react'
import { LazyWhenVisible } from './LazyWhenVisible'

export const DeferredSection = memo(function DeferredSection({
  children,
  minHeight,
  fallback,
  rootMargin,
}: {
  children: ReactNode
  minHeight?: string
  fallback?: ReactNode
  rootMargin?: string
}) {
  return (
    <LazyWhenVisible minHeight={minHeight} fallback={fallback} rootMargin={rootMargin}>
      <Suspense fallback={fallback}>{children}</Suspense>
    </LazyWhenVisible>
  )
})
```

**English:** Named exports are wrapped with `lazy(() => import(...).then(({ Named }) => ({ default: Named })))`. Below-the-fold sections split into **separate JS chunks** loaded only when scrolled near (`LazyWhenVisible`) and when the chunk is ready (`Suspense`). The hero stays eager for fast first paint.

**العربية:** `lazy` يقسّم الأقسام إلى **ملفات جافاسكربت منفصلة**. `DeferredSection` يجمع بين التحميل عند الاقتراب من الشاشة (`LazyWhenVisible`) وبين انتظار الجزء (`Suspense`). القسم الأول (Hero) يُحمَّل مباشرة لتحسين أول عرض.

## 5. Shared `IntersectionObserver` (one observer per `rootMargin`)

**Files:** `src/lib/observe-intersection.ts`

```ts
const callbacks = new Map<Element, () => void>()
const observers = new Map<string, IntersectionObserver>()

function getOrCreateObserver(rootMargin: string): IntersectionObserver {
  const cached = observers.get(rootMargin)
  if (cached) return cached
  const observer = new IntersectionObserver(
    (entries) => {
      for (const entry of entries) {
        if (!entry.isIntersecting) continue
        const el = entry.target
        const cb = callbacks.get(el)
        if (!cb) continue
        observer.unobserve(el)
        callbacks.delete(el)
        cb()
      }
    },
    { root: null, rootMargin, threshold: 0 }
  )
  observers.set(rootMargin, observer)
  return observer
}

export function subscribeIntersection(
  element: Element,
  rootMargin: string,
  onIntersect: () => void
): () => void {
  const io = getOrCreateObserver(rootMargin)
  callbacks.set(element, onIntersect)
  io.observe(element)
  return () => {
    io.unobserve(element)
    callbacks.delete(element)
  }
}
```

**English:** Instead of creating one `IntersectionObserver` per section, this pattern reuses **one observer per `rootMargin`**, maps elements to callbacks, and disconnects per target after the first intersection. That reduces memory and main-thread work.

**العربية:** بدل إنشاء مراقب لكل قسم، يُعاد استخدام **مراقب واحد لكل قيمة `rootMargin`** مع خريطة عناصر→دوال، ويُلغى ربط العنصر بعد أول تقاطع. يقلّل استهلاك الذاكرة والعمل على الخيط الرئيسي.

## 6. `content-visibility`, `contain-intrinsic-size`, reserved `minHeight` (CLS)

**Files:** `src/components/currency-landing/LazyWhenVisible.tsx`

```tsx
return (
  <div
    ref={ref}
    style={{
      minHeight: visible ? undefined : minHeight,
      contentVisibility: visible ? undefined : 'auto',
      containIntrinsicSize: visible ? undefined : '480px',
    }}
    data-lazy-root={visible ? 'ready' : 'pending'}
  >
    {visible ? children : (fallback ?? <SectionSkeleton />)}
  </div>
)
```

**English:** Before the section mounts, a **minimum height** and `content-visibility: auto` hint to the browser that off-screen content can be skipped for layout/paint work; `contain-intrinsic-size` gives an estimated size so scrollbars stay stable. Together with the skeleton, this **limits cumulative layout shift (CLS)**.

**العربية:** قبل تركيب المحتوى، يُحجز ارتفاع تقريبي ويُستخدم `content-visibility: auto` ليقلّل عمل التخطيط/الرسم للعناصر خارج الشاشة، و`containIntrinsicSize` يعطي حجماً تقديرياً للاستقرار. مع الهيكل البديل (skeleton) يُقلّل **تحرك التخطيط أثناء التحميل (CLS)**.

## 8. Theme toggle: accessible button state

```tsx
<Button

  onClick={toggleTheme}
  aria-label={isDark ? 'Switch to light theme' : 'Switch to dark theme'}
```

**English:** `aria-label` describes the **next** action; `aria-pressed` reflects toggle state; icons are `aria-hidden` because the button label is textual. This follows a **toggle button** pattern for screen readers.

**العربية:** `aria-label` يصف الإجراء؛ `aria-pressed` يعكس حالة التبديل؛ الأيقونات مخفية عن قارئ الشاشة لأن الزر له نص معنوي. هذا نمط **زر تبديل (toggle)** متاح.

## 10. CSS: Tailwind v4 `@theme`, OKLCH palette, conditional smooth scroll

**Files:** `src/index.css`

```css
@custom-variant dark (&:is(.dark *));

@theme inline {
  --font-heading: ui-sans-serif, system-ui, sans-serif;
  --font-sans: ui-sans-serif, system-ui, sans-serif;
}

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
}

.dark {
  --background: oklch(0.14 0.02 250);
  --foreground: oklch(0.985 0 0);
}

@layer base {
  @media (prefers-reduced-motion: no-preference) {
    html {
      scroll-behavior: smooth;
    }
  }
}
```

**English:** Tailwind v4 `@theme` registers design tokens. **System font stacks** avoid extra font downloads (better **FCP**). **OKLCH** gives perceptually smooth light/dark palettes. Smooth scrolling is **only** applied when the user has not requested reduced motion (`prefers-reduced-motion`).

**العربية:** `@theme` يعرّف رموز التصميم. خطوط النظام تتجنب تنزيل خطوط إضافية (تحسين **FCP**). **OKLCH** يعطي ألواناً متسقة بين الوضعين. التمرير السلس يُفعَّل فقط عندما لا يطلب المستخدم تقليل الحركة.

## 13. Entry: `StrictMode` + client `createRoot`

**Files:** `src/main.tsx`

```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>
)
```

**English:** React 18 root API enables concurrent features. `StrictMode` double-invokes certain lifecycles in development to surface side-effect bugs (no effect in production builds).

**العربية:** واجهة React 18 لإنشاء الجذر تدعم الميزات الحديثة. `StrictMode` في التطوير يعيد استدعاء بعض الدورات لكشف الأخطاء (لا يؤثر في إنتاج البناء النهائي).
